# Why Axum is Great for Building Web Services

[Axum](https://github.com/tokio-rs/axum) is a web framework built on top of [Tokio](https://tokio.rs/) and [Tower](https://github.com/tower-rs/tower). It's designed to be ergonomic, modular, and type-safe.

## Key Features

### 1. Type Safety

Axum leverages Rust's type system to catch errors at compile time:

```rust
async fn handler(
    Path(id): Path<u32>,
    Json(payload): Json<CreateUser>,
) -> Result<Json<User>, StatusCode> {
    // Your code here
}
```

### 2. Extractors

Extractors make it easy to get data from requests:

- `Path` - Extract route parameters
- `Query` - Extract query parameters
- `Json` - Parse JSON bodies
- `State` - Share application state
- `Extension` - Share arbitrary data

### 3. Middleware

Built on Tower, Axum has excellent middleware support:

```rust
let app = Router::new()
    .route("/", get(handler))
    .layer(TraceLayer::new_for_http())
    .layer(CorsLayer::permissive());
```

### 4. Performance

Axum is built on Tokio, one of the fastest async runtimes:

- Zero-cost abstractions
- Efficient connection handling
- Low memory overhead

## Example: Simple API

Here's a complete example of a simple API:

```rust
use axum::{
    routing::get,
    Router,
    Json,
};
use serde::Serialize;

#[derive(Serialize)]
struct Message {
    text: String,
}

async fn hello() -> Json<Message> {
    Json(Message {
        text: "Hello, Axum!".to_string(),
    })
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/", get(hello));

    let listener = tokio::net::TcpListener::bind("127.0.0.1:3000")
        .await
        .unwrap();
        
    axum::serve(listener, app).await.unwrap();
}
```
